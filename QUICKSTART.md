# QUICKSTART — 法律事務所パイロット向け実戦ガイド

> **先に出力を見たいだけなら `/quickstart` コマンドを打つ。**
> 同梱サンプルで 60 秒、家系図・校正・訴訟分析等を**あなたのファイル・事案
> セットアップ一切なしで** 動かせる。本ドキュメントはその後、実際の案件で
> 使うときの実戦ツアー。ここは 30 分かかる読み物。読まなくても `/quickstart`
> から入れば機能は使える。

このドキュメントは **claude-bengo を既に試して、自分の案件で使うか決めた
弁護士向け**の実戦サンプル集。実務に近い 3 シナリオ（相続・離婚・交通事故）
を端から端まで歩く。

## 前提

以下が完了していること:

- `/verify` が全項目 OK
- 案件フォルダ（`~/cases/{case-name}/` など）で Claude Code を起動している

未実施なら `RUNBOOK.md` の 15〜20 分スモークテストを先に実施する。

## シナリオ 1 — 相続案件（戸籍から相続関係説明図 + 法定相続分計算）

**状況:** 被相続人の戸籍謄本を受領した。相続人を特定し、相続関係説明図を作成、法定
相続分を計算して遺産分割協議書を起案する。

### ステップ

1. **案件フォルダに移動（事案の事前登録は不要）:**
   ```sh
   cd ~/cases/smith-v-jones  # 案件ごとに分離したいフォルダで Claude Code 起動
   ```
   機密スキルを最初に使ったときに `.claude-bengo/` が自動作成され、そのフォルダの
   監査ログ・テンプレートが蓄積される。

2. **戸籍 PDF から相続関係説明図を生成:**
   ```
   /family-tree ~/Desktop/被相続人_戸籍.pdf
   ```
   プラグインは Step 2.5 で以下 4 点を確認してくる:
   - 被相続人の特定（死亡記載が 1 名なら自動）
   - 直系尊属の描画（裁判所提出用なら不要）
   - 旧字体 vs 新字体（戸籍どおり保持が既定）
   - 補助戸籍（除籍謄本・改製原戸籍）の有無
   
   出力: `family_tree_YYYY-MM-DD.html`。ブラウザで開いて「印刷 / PDF 出力」ボタンで
   そのまま裁判所提出形式に変換できる。

3. **法定相続分を計算:**
   ```
   /inheritance-calc
   ```
   会話形式で相続人構成（配偶者+子、代襲、半血、放棄等）を入力すると、分数で正確な
   法定相続分を返す。浮動小数点の丸め誤差なし。

4. **遺産分割協議書を起案:**
   ```
   /template-install      （「遺産分割協議書」を選択）
   /template-fill 財産目録.pdf
   ```

5. **監査ログを確認:**
   ```bash
   python3 ~/.claude/plugins/claude-bengo/skills/_lib/audit.py verify
   ```
   ハッシュチェーン整合性と改ざん検知を確認する。

**所要時間の目安:** 20 ページ程度の戸籍で 5〜10 分、3 通に跨る代襲ケースで 15〜20 分。

## シナリオ 2 — 離婚案件（財産分与・養育費計算）

**状況:** 離婚調停。依頼者は妻側。夫婦の共有財産・特有財産・債務を整理し、清算金と
養育費を算出する。

### ステップ

1. **案件フォルダに移動:**
   ```sh
   cd ~/cases/{案件名}
   ```

2. **財産目録テンプレートをインストール:**
   ```
   /template-install      （「財産目録」を選択）
   ```

3. **通帳・残高証明書のデータを入力:**
   ```
   /template-fill 通帳_夫.pdf 通帳_妻.pdf 残高証明書.pdf
   ```
   Step 5.5 で抽出に曖昧点があれば確認してくる:
   - 同名口座の名寄せ戦略
   - 単位正規化（万円 / 円 / 壱百万）
   - **欠落項目は `[要確認]` で残される**（v3.3.0 以降、推測補完は禁止）

4. **財産分与を計算:**
   ```
   /property-division-calc
   ```
   対話で入力:
   - 夫・妻それぞれの収入と特有財産
   - 共有財産（不動産・預貯金・保険）と共有債務（住宅ローン等）
   - 貢献度（既定 50:50、指定可）
   
   出力: 清算金の方向（夫 → 妻 / 妻 → 夫）と金額。民法 768 条準拠。

5. **養育費を計算（子がいる場合）:**
   ```
   /child-support-calc
   ```
   令和元年改定算定方式（東京・大阪家裁）で月額を決定論的に算出。1000 円単位丸め。

**所要時間の目安:** 10〜15 分（通帳 PDF の枚数次第）。

## シナリオ 3 — 交通事故案件（損害賠償計算）

**状況:** 人身事故。依頼者は被害者。入通院慰謝料・休業損害・後遺障害逸失利益を計算
し、示談書を作成する。

### ステップ

1. **案件フォルダに移動:**
   ```sh
   cd ~/cases/{案件名}
   ```

2. **賠償額を計算:**
   ```
   /traffic-damage-calc
   ```
   対話で入力:
   - 受傷部位・傷病名
   - 入院日数・通院日数・通院期間
   - 後遺障害等級（該当する場合）
   - 事故前年収
   - 過失割合
   
   赤い本（民事交通事故訴訟損害賠償額算定基準）に基づく決定論的計算。LLM の推論では
   なく確定ロジック。

3. **示談書を作成:**
   ```
   /template-install      （「交通事故示談書」を選択）
   /template-fill 診断書.pdf 事故状況報告書.pdf
   ```

4. **請求訴状（訴訟時）:**
   ```
   /template-install      （「訴状」系テンプレートを選択）
   ```

**所要時間の目安:** 5〜10 分。

## よくあるつまづき

| 症状 | 原因・対処 |
|---|---|
| 案件フォルダが意図と違う（`/case-info` で確認） | `cd` で正しい案件フォルダへ移動してから再実行 |
| PDF の読取精度が低い | 手書き戸籍や低解像度スキャンは限界あり。300dpi+ の PDF を推奨 |
| テンプレートの入力位置がズレる | XLSX レイアウト変更時は `/template-create` で再登録 |
| `family_tree_*.html` の描画が 3 人しかない | 仕様通り。相続関係説明図は直系尊属を描画しない（民法 887 条で尊属は相続人にならないため） |
| 監査ログが端末別に分散する | 定期的に `audit.py export` で事務所管理の S3 Object Lock 等にエクスポート |
| 同一戸籍を複数回処理した痕跡が複数行残る | 正常動作。`filename_sha256` で同一性を検出できる |
