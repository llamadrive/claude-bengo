# 戸籍謄本読取ガイド

## 戸籍の構造

### 基本情報（冒頭）
- **本籍**: 戸籍の所在地（住所とは異なる）
- **筆頭者**: 戸籍の代表者（通常は夫または父）

### 身分事項欄（各人物）
各人物について以下が記載される:
- **名**: 氏名
- **生年月日**: 元号 + 年月日
- **出生**: 出生地、届出日、届出人
- **婚姻**: 婚姻日、配偶者氏名、従前戸籍
- **離婚**: 離婚日、相手方
- **養子縁組**: 縁組日、養親/養子の氏名
- **死亡**: 死亡日、死亡地
- **入籍**: 入籍日、入籍事由（婚姻、転籍等）
- **除籍**: 除籍日、除籍事由

### 従前戸籍
婚姻や転籍前の元の戸籍情報。系図を遡る手がかりとなる。

## 特殊な身分事項の処理

### 養子縁組（特別養子縁組 vs 普通養子縁組）

養子縁組には2種類あり、家族グラフへの影響が根本的に異なる。

#### 普通養子縁組
- **戸籍記載**: 「養子縁組」の身分事項として、縁組日・養親氏名が記載される。
- **家族グラフ**: 実親との親子関係は**維持**される。養親との親子関係を**追加**する。
- **エッジ属性**: `relationship_type: "adoptive"` で養親関係を区別する。
- **実親欄**: 戸籍上に実父母の氏名が引き続き記載される。

#### 特別養子縁組
- **戸籍記載**: 特別養子縁組が成立すると、実親に関する記載が**消除**され、養親があたかも実親であるかのように記載される（「民法817条の2」の記載で判別可能）。
- **家族グラフ**: 実親との親子関係エッジを**切断**する。養親のみが親として記録される。
- **エッジ属性**: `relationship_type: "biological_severed"` を実親側に付与し、法的関係の断絶を表現する。
- **注意**: 特別養子縁組の事実は、実親側の戸籍や除籍を遡らない限り判明しないことがある。

### 認知（にんち）

婚姻関係にない父が子を自分の子であると法的に認める行為。

- **戸籍記載**: 子の身分事項欄に「認知　年月日 父○○認知」と記載される。父の戸籍にも「認知」の記載がある。
- **家族グラフ**: 認知前は母のみが親ノードに接続される。認知により父ノードとの親子エッジを追加する。
- **エッジ属性**: `relationship_type: "biological"`, `recognized: true`, `recognition_date: "YYYY-MM-DD"` を付与する。
- **注意**: 認知がない非嫡出子の場合、父との法的親子関係は存在しない（戸籍上、父欄が空欄）。

### 死後認知（しごにんち）

父の死後に、裁判により父子関係が認められるケース。

- **戸籍記載**: 「死後認知　年月日 裁判確定」等の記載。
- **家族グラフ**: 認知と同様に父ノードとの親子エッジを追加するが、`posthumous: true` を付与する。
- **相続への影響**: 死後認知が確定した時点で相続権が発生するため、日付情報が重要。

### 準正（じゅんせい）

非嫡出子が、父母の婚姻により嫡出子の身分を取得すること。

- **戸籍記載**: 「準正　父母の婚姻により嫡出子となる」等の記載。認知 + 婚姻の両方が前提。
- **家族グラフ**: 既存の親子エッジの属性を `legitimated: true`, `legitimation_date: "YYYY-MM-DD"` に更新する。`status` を `"non_marital"` から `"marital"` に変更する。
- **注意**: 準正には「婚姻準正」（認知後に婚姻）と「認知準正」（婚姻後に認知）の2パターンがある。

### 離縁（りえん）

養子縁組の解消。

- **戸籍記載**: 「離縁　年月日 協議離縁届出」または「年月日 裁判確定」。
- **家族グラフ**: 養親との親子エッジを**削除**するか、`dissolved: true`, `dissolution_date: "YYYY-MM-DD"` で注釈する。相続関係の時系列判定に離縁日が必要なため、エッジ削除よりも注釈を推奨する。
- **普通養子の離縁**: 実親との関係が復活する（もともと切断されていない）。
- **特別養子の離縁**: 極めて稀。特別養子縁組の離縁は民法上厳格に制限されている。

### 氏の変更

以下のイベントにより氏（姓）が変更されることがある。身分事項欄の記載から変更を追跡する。

| イベント | 記載例 | 処理 |
|----------|--------|------|
| 婚姻 | 「婚姻により○○の氏を称する」 | 氏を配偶者の氏に変更 |
| 離婚 | 「離婚により復氏」 | 旧姓に復帰（ただし婚氏続称届がある場合は維持） |
| 養子縁組 | 「縁組により○○の氏を称する」 | 氏を養親の氏に変更 |
| 離縁 | 「離縁により復氏」 | 縁組前の氏に復帰 |
| 氏の変更許可 | 「家庭裁判所の許可により氏を○○に変更」 | 裁判所許可による任意変更 |

氏の変更は人物ノードの `name_history` 配列に時系列で記録し、各時点での正式な氏名を追跡可能にする。

## 除籍・改製原戸籍の特徴

### 除籍謄本
- 全員が除籍（婚姻、死亡、転籍等で離脱）された戸籍。
- 歴史的な情報源として重要。
- 「除籍」の朱印またはマークがある。

### 改製原戸籍
- 戸籍法改正に伴い旧形式から新形式に書き換えられる前の原本。
- 主な改製時期: 昭和32年（1957年）、平成6年（1994年）のコンピュータ化。
- 改製前の情報（離婚歴等）が新戸籍に引き継がれない場合がある。

## 戸籍形式の違いと読取上の注意

戸籍抽出における最大の課題は、形式の違いに起因する。平成6年式（コンピュータ化後）と改製原戸籍（コンピュータ化前）では、レイアウト・文字・構造が大きく異なるため、それぞれに応じた読取戦略が必要である。

### 平成6年式（コンピュータ戸籍）

平成6年（1994年）以降にコンピュータ化された戸籍。現行の戸籍の大半がこの形式である。

**特徴:**
- **横書き**: 左から右への横書きレイアウト。
- **活字体**: フォントは明朝体で統一されており、OCR 精度が高い。
- **明確なフィールドラベル**: 「【氏名】」「【生年月日】」「【父】」「【母】」等のラベルが構造化されている。
- **個人単位の記載**: 一人ずつ区切られ、身分事項欄が個人ごとにまとまっている。
- **全部事項証明書 / 個人事項証明書**: 出力形式として全員分または個人分を選択可能。

**読取時のポイント:**
- フィールドラベルをキーにしたパターンマッチングが有効。
- 「【」「】」で囲まれたラベルを検出し、その直後のテキストを値として抽出する。
- 身分事項は「出生」「婚姻」「離婚」等のイベント種別ラベルで区切られている。
- 従前戸籍は「【従前戸籍】」ラベルの後に記載される。

### 改製原戸籍（手書き・タイプ打ち）

コンピュータ化以前の戸籍。昭和32年式、大正4年式、明治31年式等がある。

**特徴:**
- **縦書き**: 右から左への縦書きレイアウト。
- **手書きまたはタイプ打ち**: 筆記体・崩し字が多く、OCR 精度が大幅に低下する。
- **列順序が異なる**: 右端に本籍・筆頭者、その左に各人の身分事項欄が続く。戸主を起点に家族が記載される。
- **戸主制**: 昭和22年改正前の戸籍では「戸主」が中心となり、「前戸主」からの家督相続の記載がある。
- **セクション構造の違い**: 明確なフィールドラベルがなく、文章形式（「何年何月何日何某ト婚姻届出」等）で記載される。
- **朱書き・取消線**: 除籍された者には朱色の×印や取消線が引かれる。
- **旧字体・異体字の多用**: 新字体への変換が必要な文字が頻出する。

**読取時のポイント:**
- 縦書き画像の場合、列方向の文字認識が必要。右から左への列順を考慮する。
- 手書きの崩し字は、前後の文脈（日付パターン、地名パターン等）から推定する。
- 身分事項は文章中の動詞・キーワード（「出生」「届出」「婚姻」「入籍」「死亡」等）で分節する。
- 「同」は直前の地名・日付の繰り返しを意味する（例: 「同所」＝前述の場所）。
- 戸主の欄に「前戸主 ○○」の記載がある場合、先代との接続に利用する。
- 改製原戸籍には改製時に移記されなかった情報（離婚歴、認知等）が残存している可能性がある。必ず改製原戸籍まで遡って確認する。

### 形式判別のヒント

| 判別基準 | 平成6年式 | 改製原戸籍 |
|----------|-----------|------------|
| 書字方向 | 横書き | 縦書き |
| 文字種 | 活字（明朝体） | 手書き or タイプ打ち |
| ラベル形式 | 【氏名】等の構造化ラベル | ラベルなし、文章形式 |
| 記載単位 | 個人ごと | 戸主を起点に家族一括 |
| 用紙 | A4横、白色 | B4縦、罫線入り用紙 |
| 戸主/筆頭者 | 筆頭者 | 戸主（昭和22年以前） |

## 分籍・転籍と従前戸籍チェーンの追跡

戸籍を遡って家族関係を網羅的に把握するには、分籍・転籍による戸籍の移動を追跡する必要がある。各戸籍に記載される「従前戸籍」を手がかりに、チェーンを辿る。

### 転籍（てんせき）

本籍地を変更すること。戸籍の内容は新本籍地に移記されるが、移記されない情報がある。

- **戸籍記載**: 「転籍　年月日 ○○県○○市○○番地から転籍届出」
- **従前戸籍の記載**: 新戸籍の冒頭に「従前戸籍　○○県○○市○○番地　○○（筆頭者名）」が記載される。
- **追跡方法**: 従前戸籍に記載された本籍地・筆頭者名で、転籍前の除籍謄本を請求する。
- **注意点**:
  - 転籍を繰り返すと、古い身分事項（離婚歴、認知等）が移記されず脱落することがある。
  - 全ての身分事項を把握するには、転籍チェーンの終端（最初の戸籍）まで遡る必要がある。

### 分籍（ぶんせき）

戸籍に記載されている成年者が、新たに自分を筆頭者とする戸籍を作ること。

- **戸籍記載**: 「分籍　年月日 分籍届出」
- **従前戸籍の記載**: 分籍後の新戸籍に「従前戸籍　○○県○○市○○番地　○○（旧筆頭者名）」が記載される。
- **追跡方法**: 従前戸籍に記載された本籍地・筆頭者名で、元の戸籍（または除籍）を請求する。元の戸籍には分籍した者の出生・入籍等の身分事項が残存している。
- **注意点**:
  - 分籍は元の戸籍から「抜ける」行為であり、元の戸籍側ではその者が除籍扱いになる。
  - 分籍後の新戸籍にはその者の身分事項しか記載されないため、親兄弟の情報は元の戸籍を参照する必要がある。

### 従前戸籍チェーンの辿り方

```
現在の戸籍
  └─ 従前戸籍: ○○市△△番地（転籍）
       └─ 除籍謄本（○○市△△番地）
            └─ 従前戸籍: □□市××番地（婚姻による新戸籍編製）
                 └─ 改製原戸籍（□□市××番地）
                      └─ 婚姻前の戸籍（実家の戸籍）
```

**手順:**
1. 現在の戸籍から「従前戸籍」欄を確認する。
2. 従前戸籍の本籍地・筆頭者名を記録する。
3. 該当する市区町村に除籍謄本または改製原戸籍を請求する。
4. 取得した戸籍の「従前戸籍」欄をさらに確認し、遡及する。
5. 従前戸籍の記載がない（＝その戸籍が最初の編製）に到達するまで繰り返す。

**よくある従前戸籍の記載パターン:**

| 記載 | 意味 |
|------|------|
| 「○○県○○市○○番地　○○」 | 転籍元の本籍地・筆頭者 |
| 「○○の戸籍より入籍」 | 婚姻・養子縁組等による入籍元 |
| 「新戸籍編製」 | 婚姻等により新たに戸籍が作成された |
| 「改製につき本戸籍編製」 | 戸籍法改正に伴う改製（改製原戸籍が存在する） |

### 市区町村の合併・名称変更への対応

従前戸籍に記載された地名が現存しないことがある。明治〜昭和期の市町村合併により地名が変更されている場合は、以下で対応する。

- 法務局の管轄一覧で旧地名から現在の市区町村を特定する。
- 合併後の市区町村が戸籍を引き継いでいるため、現在の自治体に請求する。
- 地名の旧字体（例: 「縣」→「県」、「區」→「区」）にも注意する。

## 複数戸籍の統合ルール

1. 同一人物の判定: 氏名 + 生年月日 + 本籍地で照合する。
2. 矛盾がある場合は新しい戸籍の情報を優先する。
3. 養子縁組の場合、実親と養親の両方の関係を記録する（details で区別）。
4. 離婚・再婚は時系列順に記録する。

## 実サンプルから学んだパース Tips

実戦検証で頻出した記載パターンと、イベント型の判別ルール。`fixtures/family-tree/` の
各サンプルを参照しながら読むと理解が早い。

### 旧数字（大字・壱弐参）の完全対応表

改製原戸籍（昭和32年式以前）では法務省令により日付に大字を使用する。OCR 誤認を防ぐため
全て網羅する:

| 旧字 | 新字 | 旧字 | 新字 | 旧字 | 新字 |
|------|------|------|------|------|------|
| 壱、壹 | 1 | 陸 | 6 | 百、佰 | 100 |
| 弐、貳 | 2 | 漆、七 | 7 | 千、阡 | 1000 |
| 参、參 | 3 | 捌、八 | 8 | 萬 | 10000 |
| 肆、四 | 4 | 玖、九 | 9 | | |
| 伍、五 | 5 | 拾、什 | 10 | | |

**組み合わせ例:**
- 「明治四拾参年壱月壱日」 = 明治43年1月1日
- 「昭和拾七年弐月弐拾九日」 = 昭和17年2月29日
- 「昭和参拾四年拾壱月七日」 = 昭和34年11月7日

### 身分事項 vs メタ事項の判別

戸籍の身分事項欄には**家族構成に影響するイベント**と、**戸籍管理上のメタ事項**が混在する。
後者は FlatPerson / Relationship には反映しない。

**家族構成イベント（抽出対象）:**

| 記載パターン | 抽出する関係 |
|---|---|
| 「○○年○月○日○○と婚姻届出○○受附入籍」 | spouse 関係追加 |
| 「○○年○月○日協議離婚届出」 | spouse 関係に `dissolved: true` |
| 「○○年○月○日父○○届出出生」 | parent-child 関係追加 |
| 「○○年○月○日○○と養子縁組届出」 | parent-child (adoptive) |
| 「○○年○月○日死亡」 | `deathDate` 設定 |
| 「○○年○月○日前戸主○○死亡ニ因リ家督相続届出」 | parent-child（家督）+ 被相続人候補フラグ |
| 「○○年○月○日父○○認知」 | parent-child に `recognized: true` |

**メタ事項（抽出対象外、ノート扱い）:**

| 記載パターン | 意味 |
|---|---|
| 「○○改製により新戸籍編製につき○年○月○日除籍」 | 戸籍法改正による移記、家族構成に変化なし |
| 「○○年法務省令第○号により本籍欄ヲ○○ト更正」 | 行政区画変更、地名のみ更新 |
| 「送付を受けた日○○年○月○日」 | 届出書の処理日、出生・婚姻日とは別 |
| 「○○受附」 | 市区町村が届出を受け付けた日 |

**重要:** 「除籍」という文言は理由次第で扱いが変わる:
- **死亡除籍** → `deathDate` 記録
- **婚姻除籍** → spouse 関係追加 + 本戸籍から離脱の note
- **改製除籍** → メタ事項、家族関係不変
- **転籍除籍** → 従前戸籍チェーンの手がかり

### 家督相続（旧民法、昭和22年5月3日以前）のパース

**記載例（`fixtures/family-tree/koseki-complex.pdf` より）:**
> 大正4年1月5日前戸主宗太郎死亡ニ因リ家督相続届出同月弐拾九日あらたに戸籍を編製したため本戸籍消除

**抽出ルール:**
- 前戸主（宗太郎）の `deathDate` を設定する。
- 家督相続人（幸太）に `role: "家督相続人（長男）"` を付与する。
- 前戸主と家督相続人の parent-child 関係を追加する（既に戸籍の父欄にあるが、相続の事実を強調する意味で `details: "家督相続"` を付与）。
- 家督相続は**単独相続**（長男のみ）。兄弟姉妹は相続権を持たない。
- 法定持分計算では、現行民法ではなく旧民法に基づく必要がある。`inheritance-calc` に日付を渡せば正しく判別される。

### 「姉」「妹」等の単独漢字の意味

縦書き戸籍で単独の「姉」「妹」「兄」「弟」が身分事項欄冒頭に現れた場合:

- これは**届出人との続柄**を示す（戸籍記載者ではない）。
- 例: 「姉 昭和拾七年弐月弐拾九日北田幸太ト婚姻届出同日入籍」
  → 「姉（届出をしたのは対象者の姉）」が届出人、婚姻したのは対象者本人（由紀）。
- 続柄フィールドの「二女」等とは別物。混同しない。

### 記載方向の判別フロー

PDF 読取時、以下の順で形式を判別する:

1. **「全部事項証明」「個人事項証明」** のヘッダー有無 → 平成6年式（横書き）
2. **【】で囲まれた構造化ラベル** の有無 → 平成6年式
3. **縦書きレイアウト + 手書き風字体** → 改製原戸籍
4. **戸主/前戸主の記載** → 昭和22年以前の戸籍
5. **朱書き・×印での除籍表示** → 改製原戸籍

形式により抽出戦略を切り替える（ラベルベース vs 文脈ベース）。

### 相続関係説明図の描画範囲

`@agent-format/renderer` 0.1.5 以降は SPEC § 4 の normative rule により、
`data.persons` に含まれる人物を**すべて描画する**（フィルタリングしない）。
section type は `family-graph`（旧 `inheritance-diagram` は alias として受理）。

描画範囲は**データ側で制御する**（renderer 側のフラグではない）:

- **標準書式（裁判所・法務局提出用）**: persons 配列に被相続人 + 配偶者 +
  子孫（子・孫・ひ孫...再帰的）のみを含める。先順位相続人がいない場合は
  民法 889 条に従い次順位（尊属 → 兄弟姉妹）を含める。
- **拡張書式（事案整理・相続人確定作業用）**: 戸籍から抽出された person 全員
  （祖父母・兄弟姉妹・甥姪・配偶者の親族等）を含める。

**重要:** 両書式とも `variant: "jp-court"` を使う（Japanese 法律実務の体裁が
必須のため）。jp-court plugin 0.1.1 は尊属を含む全員を jp-court スタイルで
描画するため、拡張書式でも同プラグインが適切。違いは persons 配列の中身だけ。

理由・根拠:
- 裁判所・法務局に提出する相続関係説明図（不動産登記申請添付書類）の標準形式では、
  子孫がいる場合の直系尊属・兄弟姉妹は相続人にならない（民法 887 条・889 条）ため
  含めない。これはレンダラの仕事ではなく、authoring side で決める設計に SPEC が
  移行した。
- 事案整理時は次順位相続・代襲・再代襲・相続放棄後の移行の確認のため、
  尊属込みの完全な家系が必要。

SKILL.md Step 2.5 で (a)(b) を確認し、Step 3 の persons 配列を該当範囲に絞って
組み立てる。
