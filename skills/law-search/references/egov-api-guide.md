# e-Gov 法令 API ガイド

## API エンドポイント

### 条文取得（推奨 — 常にこちらを使用する）
```
GET https://laws.e-gov.go.jp/api/1/articles;lawId={lawId};article={articleNum};
```
- 1条文のみ返す。レスポンスサイズが小さく、トークン制限の心配がない
- `articleNum` は算用数字（例: `709`, `362`）

### 法令全文取得（使用禁止 — トークン制限を超える）
```
GET https://laws.e-gov.go.jp/api/1/lawdata/{lawId}
```
- ⚠ 大きな法令（民法、会社法等）は数MBの XML を返す
- **このエンドポイントは使用しない。** 条文単位 API を使用する

### 法令一覧（使用禁止 — 数千件のレスポンス）
```
GET https://elaws.e-gov.go.jp/api/1/lawlists/1
```
- ⚠ 全法令（数千件）の一覧を返す
- **使用しない。** 下記の法令 ID マッピングを使用する

## 認証

不要。公開 API である。

## レスポンス形式

XML。WebFetch で取得後、テキストとして解析する。

## 条番号の指定方法

e-Gov API の `article` パラメータには算用数字を使用する:

| ユーザー入力 | API パラメータ |
|------------|--------------|
| 第709条 | `article=709` |
| 第398条の2 | `article=398_2` |
| 第465条の10 | `article=465_10` |

枝番号のある条文は `_` で区切る。
