# 訴訟分析レポート構造ガイド

`mcp__html-report__render_report` で訴訟分析レポートを生成する際の構成ガイド。

## 推奨レポート構成

### 1. ヘッダセクション
- タイトル: 事件名（例: 「貸金返還請求事件 分析レポート」）
- サブタイトル: 事件番号（判明している場合）

### 2. 概要セクション
- summary テキストを段落として表示
- keyPoints をリストとして表示

### 3. 統計カード
stat_cards ブロックで以下を表示:
- 文書数
- 登場人物数
- タイムライン項目数
- 主張数（認否内訳）

### 4. タイムライン
timeline ブロックで時系列イベントを表示:
- 日付順（昇順）
- category による色分けを推奨
- importance 7以上のイベントを強調表示

### 5. 登場人物
card_grid ブロックで人物プロフィールを表示:
- 名前、役割、説明
- importance 順にソート
- 役割別のアイコンまたはバッジ色

### 6. 関係図
table ブロックで関係性をマトリクス表示:
- 列: 人物1, 人物2, 関係種別, 説明

### 7. 主張と認否
table ブロックで主張を表示:

| 主張 | 当事者 | 要旨 | 根拠 | 反論 | 認否 |
|------|--------|------|------|------|------|

認否ステータスのバッジ色:
- 認める: 緑（#22c55e）
- 否認: 赤（#ef4444）
- 一部認める: 黄（#eab308）
- 不知: 青（#3b82f6）
- 不明: グレー（#9ca3af）

## スタイルプリセット

`mcp__html-report__render_report` の style パラメータには `"clean"` を使用する。法律文書レポートに適した落ち着いたデザイン。

## 注意事項

- レポートは日本語で生成する。
- フォントは明朝体またはゴシック体のシステムフォントを使用する。
- 印刷対応: A4横向きで出力可能なレイアウトを推奨する。
