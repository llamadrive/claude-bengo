# フィールドマッピングガイド

ソース文書からテンプレートフィールドへのデータ抽出パターン。

## 大きなXLSXの読取

裁判所書式は100行×14列以上で結合セルが多いものがある（例: 財産目録）。`mcp__xlsx-editor__read_sheet` を `range` パラメータなしで呼ぶと、出力がClaude Codeのコンテキスト制限を超えてエラーになる場合がある。

**対策:**
1. まず `mcp__xlsx-editor__get_workbook_info` で行数・列数を確認する
2. 50行を超えるシートは `range` パラメータで30行ずつ分割して読み取る
   - 例: `range: "A1:N30"` → `range: "A31:N60"` → `range: "A61:N99"`
3. 結合セルが多いシートは特に出力が大きくなるため、より小さい範囲（20行ずつ）で読み取る

## 一般的なフィールドタイプと抽出パターン

### 当事者情報
- **氏名**: 「原告」「被告」「申立人」「相手方」等のラベルに続く人名。姓名の間にスペースがある場合とない場合がある。
- **住所**: 「住所」「居所」等のラベルに続くテキスト。〒マーク、都道府県から始まる。
- **電話番号**: 数字 + ハイフンのパターン。全角/半角の正規化が必要。
- **生年月日**: 「生年月日」ラベル隣接。元号表記（昭和○年○月○日）が多い。

### 事件情報
- **事件番号**: 「令和○年（ワ）第○号」形式。裁判所名と組み合わせで特定。
- **裁判所名**: 「○○地方裁判所」「○○簡易裁判所」等。
- **事件名**: 「○○請求事件」「○○等請求事件」。

### 金額
- **請求金額**: 全角数字 + 「円」。3桁区切りの「，」に注意。
- **遅延損害金**: 起算日 + 利率（年○%）の組み合わせ。
- **訴額**: 印紙額算定の基礎となる金額。

### 日付
- **元号変換テーブル**:
  - 明治: 1868-1912（明治45年 = 大正元年 = 1912年）
  - 大正: 1912-1926
  - 昭和: 1926-1989
  - 平成: 1989-2019
  - 令和: 2019-

## セルアドレスと行列番号の対応

| 列アドレス | 列番号 |
|-----------|--------|
| A | 1 |
| B | 2 |
| C | 3 |
| ... | ... |
| Z | 26 |
| AA | 27 |
| AB | 28 |

行番号はそのまま使用する（1-indexed）。

## テーブルフィールドの処理

### ヘッダ行の誤書込み防止

テーブルに書き込む前に、必ず `dataStartRow` の内容を `mcp__xlsx-editor__read_sheet` で確認する。以下のパターンがある場合はヘッダ行であり、書き込んではならない:

**ヘッダ行の判別パターン:**
- 「№」「No.」「番号」が含まれる
- 「名称」「種類」「金額」「残高」「所在」「備考」等の列ラベルが含まれる
- 太字や背景色が設定されている（`format_cells` の結果で判別）
- 全セルに値が入っており、いずれも具体的なデータではなくラベルテキストである

ヘッダ行を検出した場合は `dataStartRow` を +1 して次の行に書き込む。

### 行挿入時の注意
- 挿入位置はテーブル範囲の `endRow` の直前。
- 挿入後は後続フィールドの position.row がずれるため、フィールドは row の昇順で処理する。
- 結合セルがある場合は挿入前に `mcp__xlsx-editor__unmerge_cells` で解除が必要な場合がある。

### 合計行の処理
- テーブルの最終行が合計行（SUM関数等）の場合、データ行の後ろに残す。
- 合計行の判定: セルに数式が含まれる、または「合計」「計」「小計」のラベルがある。
